# Contributing to Zap

Thank you for your interest in contributing to Zap! This document outlines how you can get involved and help make Zap better.

## Ways to Contribute

There are multiple ways you can contribute:

### 1. Report Issues

* Found a bug or something confusing? Open an issue on GitHub.
* Include a clear description, steps to reproduce, and any relevant code.

### 2. Feature Suggestions

* Have an idea to improve Zap? Suggest it via a GitHub issue.
* Be specific about the problem and how your feature would help.

### 3. Code Contributions

* Implement new features or fix bugs.
* Follow these steps:

  1. Fork the repository.
  2. Create a branch for your feature or fix: `git checkout -b feature-name`.
  3. Make your changes.
  4. Write tests for your changes.
  5. Commit with a clear message.
  6. Push to your fork and create a pull request.

### 4. Documentation

* Help improve documentation and examples.
* Clear documentation makes Zap easier for everyone.

### 5. Discussion & Feedback

* Join discussions on GitHub to share your thoughts on language design and improvements.

## Guidelines

* **Be respectful**: Zap welcomes contributors from all backgrounds. Treat everyone with respect.
* **Keep it clean**: Ensure your code is readable, documented, and tested.
* **Communicate clearly**: Provide detailed explanations when reporting issues or proposing changes.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Be kind, collaborative, and constructive.

## Getting Started

* Clone the repository: `git clone https://github.com/thezaplang/zap.git`
* Check out the [README](README.md) for setup and building instructions.
* Start small and iterate.

Thank you for helping make Zap a better programming language!
